o projeto foi feito primeiro criando uma pasta no computador e iniciando um projeto node com o comando npm init para gerar o package json depois instalei o express para gerenciar as rotas da api e criei o arquivo app js como arquivo principal onde configurei o servidor para aceitar dados no formato json usando um middleware específico como não havia banco de dados real utilizei três arrays globais chamados filmes usuarios e favoritos para simular o armazenamento de dados de forma volátil o que significa que as informações se apagam se o servidor for desligado para a gestão de filmes e usuarios criei rotas de listagem cadastro edição e exclusão garantindo que cada item recebesse um id único gerado pelo comando date now no sistema de favoritos a lógica foi baseada em relacionar o id do usuário com o id do filme dentro de um array próprio validando sempre se os ids envolvidos realmente existiam antes de confirmar a operação a api funciona respondendo requisições na porta 3000 e utiliza métodos como find e filter para manipular os dados dos arrays devolvendo mensagens de erro ou sucesso acompanhadas dos códigos de status http corretos para cada situação o arquivo deve se chamar readme.md e o tipo dele é markdown
